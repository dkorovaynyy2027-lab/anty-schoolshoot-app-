# College Alert (eBezpeka)

Проста та швидка система локального оповіщення про надзвичайні ситуації для навчальних закладів. Система дозволяє студентам швидко повідомляти про небезпеку, а адміністрації — миттєво оголошувати загальну тривогу.

## Як це працює загалом?

Система поділена на дві логічні частини (ролі): **Студент** (звичайний користувач) та **Адміністратор** (охоронець, черговий або керівництво).

1. **Хтось помічає небезпеку:** Студент у своєму додатку обирає корпус і поверх, де він знаходиться, і затискає велику кнопку SOS на екрані.
2. **Адмін отримує запит:** На екрані адміністратора миттєво з'являється червоне вікно з точною локацією, звідки надійшов сигнал. Адмін може оцінити ситуацію та підтвердити тривогу або скасувати її (щоб уникнути помилкових чи жартівливих спрацювань).
3. **Загальна тривога:** Якщо адмін натискає "Підтвердити", сервер автоматично розсилає команду "ТРИВОГА" на всі телефони студентів, які зараз підключені до системи.
4. **Реакція додатку:** Телефони студентів починають вібрувати, на екрані з'являється червоне попередження "Терміново в укриття!" та вилазить системне сповіщення (банер).
5. **Відбій:** Коли все спокійно, адмін тисне "Відбій тривоги", і екрани студентів повертаються у звичайний, безпечний стан.

## Технічна сторона (Стек)

- **Клієнт (Додаток):** Написаний на .NET MAUI (C#, XAML). Це означає, що один і той самий код працює і на Android, і на iOS.
- **Сервер:** Написаний на ASP.NET Core. Він виступає центральним мозком, який тримає зв'язок між усіма додатками.
- **Зв'язок (SignalR):** Ми використовуємо технологію WebSockets через бібліотеку SignalR. Замість того, щоб додаток постійно питав сервер "чи є тривога?", додаток просто тримає відкриту "трубку". Сервер сам миттєво "кричить" у цю трубку, коли настає небезпека. Це забезпечує затримку в мілісекунди.

## Важливе обмеження: Робота у фоні

Оскільки це локальна система, яка не використовує платних серверів Apple/Google, є фундаментальна різниця в тому, як телефони ведуть себе, коли ви згортаєте додаток (натискаєте Home):

- **На Android:** Система дозволяє тримати сокет (SignalR) відкритим у фоні трохи довше. Сповіщення та вібрація скоріше за все спрацюють, навіть якщо ви нещодавно згорнули додаток.
- **На iPhone (iOS):** Apple має дуже жорстку систему енергозбереження. Як тільки ви згортаєте додаток, iOS дає йому 3-5 секунд і після цього **повністю "відрізає" йому інтернет**. Додаток фізично не може почути сигнал тривоги від сервера. Щоб "будити" заблокований iPhone, потрібна технологія Apple Push Notifications (APNs), яка вимагає платного акаунта розробника Apple ($99/рік). Наразі, без цієї ліцензії, додаток на iOS має бути відкритим, щоб миттєво отримати сигнал.

## Як запустити?

Система працює локально (телефон і комп'ютер мають бути підключені до одного Wi-Fi).

### 1. Запуск сервера (робити першим)
Відкрийте термінал, перейдіть у папку сервера і запустіть його:
```bash
cd Server/CollegeAlert.Server
dotnet run
```
*Увага: у файлі `AlertService.cs` в додатку перевірте, щоб була вказана локальна IP-адреса вашого Mac/PC (наприклад `http://192.168.0.102:5050`).*

### 2. Запуск на Android
```bash
dotnet build CollegeAlert.csproj -t:Run -f net10.0-android
```

### 3. Запуск на iPhone (через кабель з Mac)
```bash
dotnet build CollegeAlert.csproj -f net10.0-ios -p:RuntimeIdentifier=ios-arm64 -p:ApplicationId=ua.college.CollegeAlert
dotnet build CollegeAlert.csproj -t:Run -f net10.0-ios -p:RuntimeIdentifier=ios-arm64 -p:ApplicationId=ua.college.CollegeAlert -p:_DeviceName="Ім'я вашого iPhone"
```
