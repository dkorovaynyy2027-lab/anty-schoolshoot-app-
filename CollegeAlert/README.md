## 📖 Що це таке?
**College Alert** — це закрита локальна система безпеки. Вона дозволяє будь-якому студенту миттєво сповістити охорону про небезпеку, а адміністрації — в один клік підняти загальну тривогу на всіх телефонах у закладі. Система спроектована так, щоб працювати в локальній мережі Wi-Fi, без потреби у зовнішніх серверах.

## 🛠 Як це працює?

Система працює у реальному часі (затримка становить лічені мілісекунди) і має дві ролі:

### 1. 🎓 Студент (Користувач)
- Коли виникає небезпека, студент відкриває додаток.
- Обирає свій корпус та поверх.
- **Затискає велику кнопку SOS**. 
- Все. Сигнал миттєво летить на пульт адміністратора.

### 2. 🛡 Адміністратор (Охорона/Керівництво)
- На екрані адміністратора з'являється червоне сповіщення з точною вказівкою, де саме натиснули кнопку.
- Охорона оцінює ситуацію. Якщо загроза реальна — натискає **«Підтвердити»**.
- Сервер автоматично розсилає сигнал тривоги на **всі** телефони студентів. 
- На екранах студентів вмикається червоний інтерфейс, телефони починають вібрувати та показують системне сповіщення із закликом пройти в укриття.
- Коли небезпека минає, адміністратор натискає **«Відбій тривоги»**, і додатки студентів повертаються у звичайний стан.

## 🚀 Технологічний стек
Проект складається з двох основних частин:

- **📱 Мобільний додаток (Клієнт):** Написаний на **.NET MAUI**. Один код ідеально працює як на Android, так і на iPhone.
- **🖥 Сервер:** Написаний на **ASP.NET Core**. Виступає «мозком», що обробляє всі сигнали.
- **⚡ Зв'язок:** Використовується **SignalR** (WebSockets). Додатки тримають постійне з'єднання із сервером. Це дозволяє серверу «штовхати» команди на телефони без жодних затримок, що критично важливо під час надзвичайних ситуацій.
- **🔔 Сповіщення:** Інтегрований `Plugin.LocalNotification` для системних банерів та звукових сигналів.

## ⚠️ Важливо: Особливості роботи у фоні

Система є **локальною**, і вона не підключена до глобальних платних сервісів Apple (APNs) або Google (FCM). Через це поведінка згорнутих додатків відрізняється:

- 🤖 **Android:** Android дозволяє фоновим процесам "жити" довше. Якщо ви згорнете додаток, він ще певний час зможе приймати сигнали через відкритий сокет і вмикати вібрацію та банери.
- 🍏 **iOS (iPhone):** iOS має надзвичайно жорсткі правила збереження заряду батареї. Як тільки ви натискаєте кнопку Home (свайпаєте вгору), iOS **за лічені секунди примусово розриває інтернет-з'єднання** для додатку. 
  > *Сплячий iPhone фізично не почує сервер без Apple Push Notifications.* 
  У поточній версії, додаток на iOS має бути **відкритим** на екрані, щоб гарантовано отримати сигнал тривоги.

## 💻 Інструкція із запуску

Для розгортання вам знадобиться локальна мережа Wi-Fi. Комп'ютер-сервер та всі телефони мають бути підключені до неї.

### Крок 1. Запуск Сервера
Відкрийте термінал, перейдіть до папки сервера та запустіть його (він слухатиме порт `5050`):
```bash
cd Server/CollegeAlert.Server
dotnet run
```
📌 *Перед компіляцією додатку не забудьте у файлі `AlertService.cs` вказати локальну IP-адресу вашого комп'ютера-сервера (наприклад: `http://192.168.0.102:5050`).*

### Крок 2. Встановлення на Android
Підключіть Android-пристрій через USB (з увімкненим режимом розробника) і виконайте:
```bash
dotnet build CollegeAlert.csproj -t:Run -f net10.0-android
```

### Крок 3. Встановлення на iOS
Підключіть розблокований iPhone до вашого Mac. Переконайтеся, що ви довіряєте комп'ютеру:
```bash
# Збірка
dotnet build CollegeAlert.csproj -f net10.0-ios -p:RuntimeIdentifier=ios-arm64 -p:ApplicationId=ua.college.CollegeAlert

# Встановлення та запуск
dotnet build CollegeAlert.csproj -t:Run -f net10.0-ios -p:RuntimeIdentifier=ios-arm64 -p:ApplicationId=ua.college.CollegeAlert -p:_DeviceName="Ім'я вашого iPhone"
```
